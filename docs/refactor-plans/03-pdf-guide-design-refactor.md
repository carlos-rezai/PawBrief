## Problem Statement

The current `SinglePDF.tsx` and `MergedPDF.tsx` implementations diverge significantly from
the Sienna prototype designs in `src/assets/design_handoff_pawbrief/`. The gaps span
structural layout, content rendering, typography/colour decisions, and at least one
rendering bug. A visual comparison of the prototype HTML files against the live PDF
preview confirms every item below.

**Structural:** The Emergency Callout is implemented as a numbered `GSection` (flat list)
instead of its own bordered-box component with a "IN AN EMERGENCY" header strip and
vet / contacts two-column split. This off-by-one cascades to all section numbers — Feeding
shows as n=2 but should be n=1.

**Content missing or wrong:** Food cards show `brand` as title and `flavor` as subtitle with
`texture` as a separate Tag — the prototype combines them as `"brand · flavor"` title with
`texture` as subtitle. Toy descriptions are dropped. Dietary notes are never rendered.
Feeding, Favourites, and Health sections lack their left/right two-column layouts and eyebrow
labels. The Routine section shows only the clock with no slot legend. Comfort items and
Favourite spots are plain `Text` nodes instead of accent-variant Tags. Medication cards show
only name and dosage, missing frequency and instructions.

**Colour / style errors:** Good to Know note left-borders use `colors.primary` (sienna)
instead of `colors.accent` (olive). Allergies text is unstyled. The footer is truncated to
"Made with PawBrief" with no contextual copy and no generation date.

**Cover Band incomplete:** The "CARE GUIDE" eyebrow label, Wordmark (Mark + "PawBrief"
text), and `PawBriefMark` fallback when no photo are all absent. Avatar diameter is 72 px
instead of the specified 92 px (58 px in the merged guide).

**Rendering bug:** `RoutineClock` renders its track ring and time-position labels but draws
no arc paths for any activity slots. The arcs are generated by the `arcPath` utility but
do not appear in the PDF output. Root cause is unknown and must be investigated.

**MergedPDF-specific:** `HealthCol` and `NotesCol` return `null` instead of `<NotAdded />`
when a column has no data, breaking row alignment. The merged footer copy does not name the
cats.

---

## Solution

Refactor both `SinglePDF.tsx` and `MergedPDF.tsx` to match the prototype 1:1, in a
sequence of small, independently working commits. Each commit targets one visual concern
and leaves both PDFs rendering correctly throughout.

The sequence proceeds in dependency order: simple property fixes first (food card format,
border colours, footer copy), then structural rewrites (EmergencyCallout sub-component +
section renumbering, Cover Band), then section-level layout additions (Feeding two-column,
Routine legend, Favourites two-column, Health two-column). The RoutineClock arc rendering
bug is fixed as a standalone commit before the legend is added.

No data model changes. No new wizard steps. One new sub-component: `EmergencyCallout.tsx`.

## Commits

### Commit 1 — Fix food card format and toy description in both PDFs

In the Feeding section of both PDFs, `MiniCard` currently receives `title={entry.brand}`
and `subtitle={entry.flavor}`, with `entry.texture` as a separate Tag. The prototype
renders `"brand · flavor"` as the card title and `texture` as the subtitle. Update both
`FeedingCol` (MergedPDF) and the inline Feeding block (SinglePDF) to pass
`title={[entry.brand, entry.flavor].filter(Boolean).join(' · ')}` and
`subtitle={entry.texture}`. Remove the standalone texture Tag.

In the Favourites section of both PDFs, `ToyEntry` MiniCards only pass `title={t.name}`,
dropping `t.description`. Update to also pass `subtitle={t.description}` when present.

Update affected tests to match the new rendered text.

### Commit 2 — Fix Good to Know note left-border colour in both PDFs

The note item `borderLeftColor` uses `colors.primary` (sienna) in both PDFs. The prototype
specifies the olive accent colour. Change `colors.primary` → `colors.accent` for the note
left-border style in `SinglePDF` and `NotesCol` in `MergedPDF`. No test changes needed.

### Commit 3 — Fix footer copy and add generation date in both PDFs

The footer currently renders only "Made with PawBrief". Update:

- `SinglePDF`: `"Made with PawBrief · keep this handy while caring for {basics.name}"`
- `MergedPDF`: `"Made with PawBrief · caring for {a.name} & {b.name}"`

Both footers gain a right-side generation date using
`new Date().toLocaleDateString('en-GB', { day: 'numeric', month: 'short', year: 'numeric' })`.

Update footer tests to assert the new copy.

### Commit 4 — Fix HealthCol and NotesCol to show NotAdded in MergedPDF

`HealthCol` returns `null` when `medical` is absent or has no health data. `NotesCol`
returns `null` when there are no special notes. Both should return `<NotAdded />` so that
row alignment is preserved when one cat has data and the other does not.

Add tests: "shows 'Not added' in Health column when one cat has no health data" and
"shows 'Not added' in Notes column when one cat has no notes".

### Commit 5 — Extract EmergencyCallout sub-component + fix section numbering

## Decision Document

**EmergencyCallout as a standalone sub-component** — extracted because both guides need it
and the rendering differs (single guide has a fixed vet column; merged guide has the
shared-vet variant). A single component with a `sharedVet` flag avoids duplicating the box
structure in both files.

**Section numbering reset to n=1..5** — Emergency Callout is not a numbered section in the
prototype; it is a standalone callout above the numbered body. All five numbered sections
(Feeding, Routine, Favourites, Health, Good to Know) are renumbered 1–5, with Health and
Good to Know conditionally omitted as before.

**Feeding servings rendered as Tags under a "SERVINGS" eyebrow** — the prototype's
`amountGrams` + `feedingTimes` model was superseded by `ServingEntry { time, grams }` as
decided in the design log (Q5). The serving Tags `"07:30 · 70g"` carry both pieces of
information. A "SERVINGS" eyebrow keeps the section readable without needing a KV row.

**KV component left in place but unused** — the prototype used `KV` for "Per serving" and
"Times" rows in Feeding, but the merged `ServingEntry` model makes those rows unnecessary.
`KV` is already built and tested; removing it is out of scope.

**`formatRange` utility in `src/utils/`** — the routine legend needs `"HH:MM–HH:MM"`
formatted ranges. Pure function with no side effects belongs in `utils/` with a test.
Handles midnight wrap.

**Routine legend layout differs between single and merged guides** — the single guide has
sufficient horizontal space for clock left / two-column legend right (matching the
prototype). The merged guide columns are too narrow; each column stacks clock then legend
vertically, matching `pb-guide-compare.jsx`.

**`MergedPDF.COVER_PHOTO_SIZE` corrected to 58** — the design specifies 58×58 px circles in
the merged cover band vs 92 px in the single guide. The constant was misnamed and
incorrectly valued (72); fix to 58.

**Toy description rendered as MiniCard subtitle** — `ToyEntry.description` is optional.
When present it provides useful context. The prototype passes it as `sub` to MiniCard.

**RoutineClock arc rendering** — the exact fix depends on the root cause found during
investigation in Commit 8. The most likely candidates are a prop-name difference in
react-pdf's SVG implementation or a `fill="none"` handling constraint. The fix will be
committed with a comment referencing the root cause once confirmed.

## Testing Decisions

**What makes a good test:** PDF components use a `@react-pdf/renderer` mock that maps
primitives to plain HTML equivalents. Good tests assert that data the pet-sitter needs is
present and readable — text content, link hrefs, image sources. Tests do not assert pixel
positions, flex directions, or style property values.

**Modules tested:**

- `SinglePDF` — extended in each commit as new content is added
- `MergedPDF` — extended in each commit; Commit 4 adds the missing NotAdded column tests
- `EmergencyCallout` — dedicated tests: "IN AN EMERGENCY" header, vet name/phone/Maps link,
  contacts, shared-vet variant renders vet once full-width
- `RoutineClock` — Commit 8 adds assertion for "A DAY" centre label
- `formatRange` — unit tests in `src/utils/`: normal ranges, midnight wrap, sub-hour
  durations

**Prior art:** `SinglePDF.test.tsx` and `MergedPDF.test.tsx` use the
`vi.mock("@react-pdf/renderer")` pattern. All new tests follow the same structure.

## Out of Scope

- Photo rendering in PDFs (avatar, plating photo, note photos) — already wired; no changes
- Dark mode or theme switching in PDFs — react-pdf has no dynamic theming
- Photo cropping or aspect-ratio adjustments in the Cover Band
- Multi-page pagination controls — react-pdf handles wrapping automatically
- Any wizard step changes
- Any Dashboard or Preview page changes
- Deleting the unused `KV` sub-component

## Further Notes

The prototype uses a slightly different data shape (`amountGrams`, `feedingTimes`,
`activitySlots`, `supplements`, `toys`, `treats`) compared to our production types
(`ServingEntry`, `slots`, `supplementEntries`, `toyEntries`, `treatEntries`). Wherever the
prototype's field names differ, this plan uses the production type field names and structures
described in `docs/ubiquitous-language.md`.

The `dietaryNotes` field on `FeedingData` is typed `string | undefined`. Treat it as present
only when non-empty (same guard as the existing `platingInstructions` check).
